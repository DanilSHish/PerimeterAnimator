## Обзор

PerimeterAnimator использует простую двухформенную архитектуру:

- **MainForm** — Главное окно с холстом анимации и элементами управления
- **ParametersForm** — Модальный диалог для настройки параметров анимации

Приложение построено на событийно-ориентированной архитектуре Windows Forms с GDI+ для 2D-графики.

## Структура классов

### MainForm

**Обязанности:**
- Отображение холста анимации (элемент управления Panel)
- Управление состоянием анимации (запущена/остановлена)
- Обработка отрисовки через события `Paint`
- Управление обновлениями анимации по таймеру

**Ключевые члены:**

```cpp
private:
    int lineLength;      // Длина анимированной линии
    int squareSize;      // Размер периметра анимации
    int currentPosition; // Текущая позиция головы (0..maxPosition)
    int intervalMs;      // Интервал таймера в миллисекундах
    Color lineColor;     // Цвет линии для отрисовки
    int maxPosition;     // Максимальная позиция (длина периметра)

    void StartStop(bool start); // Управление состоянием анимации
```

**Ключевые методы:**

| Метод | Описание |
|-------|----------|
| `panel1_Paint()` | Отрисовка линии через GDI+ Graphics::DrawLine |
| `timer1_Tick()` | Обновление позиции и запуск перерисовки |
| `StartStop()` | Переключение состояния анимации, обновление UI |

### ParametersForm

**Обязанности:**
- Сбор пользовательского ввода для параметров анимации
- Валидация ограничений ввода
- Возврат параметров в MainForm через геттеры

**Ключевые члены:**

```cpp
private:
    int lineLength;   // Выбранная длина линии
    int squareSize;   // Выбранный размер квадрата
    int intervalMs;   // Выбранный интервал таймера
    Color lineColor;  // Выбранный цвет линии
```

**Публичный интерфейс:**

```cpp
int GetLineLength()    { return lineLength; }
int GetSquareSize()    { return squareSize; }
int GetIntervalMs()    { return intervalMs; }
Color GetLineColor()   { return lineColor; }
```

## Алгоритм анимации

Линия движется по часовой стрелке по периметру квадрата:

```
(0,0) ──────────────────► (size,0)
   │                        │
   │      PERIMETER         │
   │   (по часовой стрелке) │
   ▼                        ▼
(0,size) ◄──────────────── (size,size)
```

### Сопоставление направлений

| Направление | Сторона | Диапазон координат |
|-------------|---------|-------------------|
| 0 | Верх | x: 0 → size-1, y: 0 |
| 1 | Право | x: size-1, y: 0 → size-1 |
| 2 | Низ | x: size-1 → 0, y: size-1 |
| 3 | Лево | x: 0, y: size-1 → 0 |

### Вычисление позиции

```cpp
// Определяем, на какой стороне квадрата находится позиция
if (currentPosition < maxCoord) {
    direction = 0; // Верхняя сторона
} else if (currentPosition < 2 * maxCoord) {
    direction = 1; // Правая сторона
    sidePosition %= maxCoord;
} else if (currentPosition < 3 * maxCoord) {
    direction = 2; // Нижняя сторона
    sidePosition %= maxCoord;
} else {
    direction = 3; // Левая сторона
    sidePosition %= maxCoord;
}
```

### Отрисовка линии

Линия может огибать несколько сторон. Алгоритм:

1. Начать с `currentPosition`
2. Нарисовать отрезок на текущей стороне до угла или конца линии
3. Если осталась длина, продолжить на следующей стороне
4. Повторять, пока не нарисовано `lineLength` пикселей

```

────────────────────────────────────────────────────────────────────────────

                                                      ↓
Нажать           ParametersForm →    DialogResult = OK
"Применить"          (возврат в MainForm)
                                                      ↓

```

## Паттерны проектирования

### 1. Разделение ответственности (Separation of Concerns)

- **UI-слой**: Windows Forms обрабатывают взаимодействие с пользователем
- **Слой логики**: Классы форм управляют состоянием анимации
- **Слой отрисовки**: GDI+ выполняет фактическую отрисовку

### 2. Паттерн модального диалога

```cpp
// MainForm открывает ParametersForm как модальный диалог
ParametersForm parametersForm(panel1->Width, panel1->Height);
if (parametersForm.ShowDialog() == System::Windows::Forms::DialogResult::OK) {
    // Извлекаем параметры только при OK
    lineLength = parametersForm.GetLineLength();
    // ... и т.д.
}
```

### 3. Конечный автомат (неявный)

Состояния анимации: `STOPPED` → `RUNNING` → `STOPPED`

Переходы между состояниями управляются методом `StartStop(bool)`.

### 4. Двойная буферизация

Элемент управления Panel в Windows Forms использует двойную буферизацию по умолчанию, предотвращая мерцание во время анимации.

## Производительность

1. **Invalidate() вместо Refresh()** — использует WM_PAINT для плавной анимации
2. **Минимум аллокаций** — Pen создаётся каждый кадр (можно оптимизировать)
3. **Фиксированный размер буфера** — нет динамической памяти в цикле анимации
